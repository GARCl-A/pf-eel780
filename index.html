Relato do Cliente

"Tenho uma loja de ração e o sistema que uso atualmente não atende algumas das minhas necessidades. Gostaria de conseguir gerar uma lista de clientes por algum filtro, como um produto específico ou um conjunto de produtos e enviar para os números cadastrados dos clientes que costumam comprar esse tipo de produto um aviso com as promoções."

Proposta de Sistema

Uma solução abrangente para gestão comercial, concentrando-se em um banco de dados central para produtos, categorias, clientes, vendas e estoque. Cadastros detalhados de produtos, que serão harmonizados com informações de clientes para personalização. A integração entre vendas e estoque permitirá análises estratégicas, alertas automáticos evitarão escassez de estoque e interações proativas informarão clientes sobre descontos e produtos em baixa, impulsionando sua eficiência operacional e satisfação do cliente.

Valor Entregue

Nosso sistema economiza tempo e é feito para organizar. Ele otimiza sua rotina ao enviar mensagens automaticamente para seus clientes, economizando seu tempo valioso. Além disso, oferece um arsenal de ferramentas para manter tudo perfeitamente organizado, garantindo que você esteja sempre no controle.

Casos de Uso

Geração de Lista de Clientes e Envio de Promoções

Ator Principal: Dono da loja de ração

Descrição: Este caso de uso permite ao dono da loja de ração gerar uma lista de clientes com base em filtros específicos, como produtos ou categorias de produtos, e enviar promoções para os clientes selecionados.

Fluxo Principal:

  1. O dono da loja acessa o sistema de gestão comercial.
  2. Ele seleciona a opção "Geração de Lista de Clientes e Envio de Promoções".
  3. O sistema exibe opções de filtros, como produtos específicos ou categorias de produtos.
  4. O dono da loja escolhe os filtros desejados e define os critérios para a seleção de clientes.[A4]
  5. O sistema gera uma lista de clientes com base nos filtros e critérios selecionados.
  6. O dono da loja revisa a lista de clientes gerada.
  7. Ele compõe uma mensagem promocional ou seleciona uma promoção existente para enviar.[A7]
  8. O sistema envia a mensagem promocional para os clientes selecionados através dos números cadastrados.
  9. O dono da loja recebe uma confirmação de que as mensagens foram enviadas com sucesso.

Caminhos Alternativos:

A4- O dono da loja não selecionou nenhum filtro.

  1. O sistema gera a lista com todos os clientes cadastrados.

A7- O dono da loja deseja não enviar uma mensagem promocional.

  1. O dono da loja clica no botão 'salvar relatório'.
  2. O sistema gera um arquivo com a lista de clientes sem enviar para os contatos.

Cadastro de Novo Cliente

Ator Principal: Atendente da loja de ração

Descrição: Este caso de uso permite ao atendente da loja de ração cadastrar um novo cliente no sistema.

Fluxo Principal:

  1. O atendente acessa o sistema de gestão comercial.
  2. Ele seleciona a opção "Cadastro de Novo Cliente".
  3. O sistema exibe um formulário de cadastro de cliente.
  4. O atendente preenche as informações do cliente, que incluem:
    • Nome completo
    • Endereço
    • Número de telefone
    • E-mail (opcional)
    • Informações sobre animais de estimação (se aplicável)
  5. O atendente verifica se todas as informações obrigatórias foram preenchidas corretamente.
  6. Se todas as informações estiverem corretas, o atendente confirma o cadastro.[A6]
  7. O sistema armazena as informações do novo cliente no banco de dados.[A7]
  8. O atendente recebe uma confirmação de que o cliente foi cadastrado com sucesso.[A8]
  9. O cliente recebe em seu telefone uma mensagem de boas-vindas em seu número de celular.

Caminhos Alternativos:

A6- O atendente identifica informações em falta ou incorretas.

  1. O atendente corrige os campos necessários e tenta confirmar novamente.

A7- O sistema encontra algum erro ao armazenar as informações.

  1. O sistema informa ao atendente sobre o problema, exibindo a mensagem de erro adequada.

A8- O sistema já possuí um cliente com o mesmo nome ou número de telefone.

  1. O atendente é notificado com o sistema, que exibe os clientes duplicados.[A8.A]
  2. O atendente verifica se existe uma duplicata do cliente no sistema.[A8.B]
    • A8.A- O atendente escolhe qual duplicata será armazenada no sistema.
    • A8.B- O atendente escolhe manter os dois.

Realização de Venda

Ator Principal: Atendente da loja de ração

Descrição: Este caso de uso permite ao atendente da loja de ração registrar uma venda, associando os produtos comprados ao perfil do cliente, a fim de enriquecer o histórico de compras do cliente.

Fluxo Principal:

  1. O atendente acessa o sistema de gestão comercial.
  2. Ele seleciona a opção "Realização de Venda".
  3. O sistema exibe um formulário de venda, que inclui as seguintes etapas:
    • Identificação do cliente (pesquisa pelo nome, número de telefone ou código de cliente).[A3]
    • Seleção dos produtos a serem vendidos, incluindo quantidade e preço unitário.
    • Cálculo do valor total da venda.
    • Registro da forma de pagamento (dinheiro, cartão de crédito, etc.).
    • Confirmação da venda.
  4. O atendente completa as etapas do formulário de venda.
  5. O sistema associa os produtos comprados ao perfil do cliente identificado.[A5]
  6. O sistema atualiza o estoque de produtos de acordo com a venda realizada.[A6]
  7. O sistema armazena as informações da venda no banco de dados, incluindo a data e os produtos vendidos.
  8. O atendente emite um comprovante da venda para o cliente.
  9. O atendente encerra a transação.

Caminhos Alternativos:

A3- O atendente não encontra o cliente no sistema.

  1. O atendente decide se cadastra um novo cliente ou continua o atendimento sem cadastro.

A5- O sistema tem problemas na associação dos produtos ao perfil do cliente

  1. O sistema exibe um alerta para o atendente com a mensagem de erro adequada.

A6- A venda excede a quantidade disponível em estoque de algum produto.

  1. O sistema notifica o atendente sobre a possível falta de estoque e permite que ele ajuste a venda.

Alerta de Estoque Baixo

Ator Principal: Sistema

Descrição: Este caso de uso descreve como o sistema identifica automaticamente quando um produto atinge um nível de estoque baixo e notifica o dono da loja sobre a necessidade de reabastecimento.

Fluxo Principal:

  1. O sistema monitora constantemente os níveis de estoque de produtos cadastrados no banco de dados.
  2. Em intervalos regulares ou quando uma venda é registrada, o sistema verifica o estoque atual de cada produto.
  3. Se o estoque de um produto atingir ou cair abaixo de um limite de estoque baixo predefinido, o sistema gera um alerta.
  4. O sistema notifica o dono da loja por meio de um aviso ou mensagem, informando quais produtos têm estoque baixo.
  5. O dono da loja recebe a notificação.
